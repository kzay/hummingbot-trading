# Shared Market History

## Objective

Provide one bounded market-history contract that runtime, UI, and analytics can read without turning telemetry or UI caches into canonical market data.

## Ownership

- Canonical market bars are written by the data plane through `services/ops_db_writer/main.py`.
- `market_bar_v2` is the canonical derived bar store for shared history reads.
- `realtime_ui_api` is a read-model consumer only.
- `minute.csv` remains bot telemetry and operator evidence, not canonical market history.

## Source Families

- `quote_mid`: primary canonical market-history source.
- `exchange_ohlcv`: secondary labeled source for exchange candle reads.
- Non-canonical fallbacks:
  - `minute.csv`
  - UI in-memory stream caches
  - ad hoc REST backfill

## Runtime Behavior

- `MidPriceBuffer` continues to build live bars from `add_sample()`.
- When `HB_HISTORY_SEED_ENABLED=true`, runtime attempts a one-shot preload from `market_bar_v2` before the first live sample.
- Runtime seed acceptance is governed by `HistoryPolicy`-style env controls:
  - `HB_HISTORY_SOURCE_PRIORITY`
  - `HB_HISTORY_ALLOW_FALLBACK`
  - `HB_HISTORY_RUNTIME_MIN_STATUS`
  - `HB_HISTORY_RUNTIME_MIN_BARS`
  - `HB_HISTORY_MAX_ACCEPTABLE_GAP_S`
- If preload fails or quality is insufficient, runtime falls back to the existing live warmup path without crashing.

## UI Behavior

- `HB_HISTORY_UI_READ_MODE=legacy` keeps the existing candle path.
- `HB_HISTORY_UI_READ_MODE=shadow` computes provider output and parity metadata while still serving the legacy path.
- `HB_HISTORY_UI_READ_MODE=shared` serves provider-backed candles with explicit quality metadata and compatibility fallback.

## Storage Bounds

- `market_bar_v2` stores 1-minute bars keyed by `(connector_name, trading_pair, bar_source, bar_interval_s)`.
- Rolling per-key caps are enforced with `OPS_DB_MARKET_BAR_V2_RETENTION_MAX_BARS`.
- Distinct-key and storage-budget evidence is tracked with:
  - `OPS_DB_MARKET_BAR_V2_MAX_DISTINCT_KEYS`
  - `OPS_DB_MARKET_BAR_V2_STORAGE_BUDGET_MB`
- Operator evidence is generated by:
  - `scripts/ops/backfill_market_bar_v2.py`
  - `scripts/ops/report_market_bar_v2_capacity.py`
- Multi-minute bars are rolled up at read time by the provider.

## Validation Focus

- Dual-write parity between `market_quote_bar_minute` and `market_bar_v2`.
- `MidPriceBuffer.seed_bars()` indicator parity with live-fed bars.
- UI parity in shadow mode before shared cutover.
