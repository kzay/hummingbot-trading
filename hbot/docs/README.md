# Documentation Hub

## Purpose
Centralized, maintainable documentation for infrastructure, architecture, strategy, risk, financial controls, validation, and operations.

## Scope
This hub is the source of truth for how the project is built, operated, and governed.

## Sections
- `infra/`: deployment topology, compose profiles, env and secrets.
- `techspec/`: service contracts, interfaces, technical behavior.
- `architecture/`: system and data-flow views.
- `financial/`: fee assumptions, KPI and budget limits.
- `strategy/`: EPP and ML strategy behavior.
- `risk/`: policy hierarchy and kill-switch taxonomy.
- `ops/`: runbooks and incident response.
- `validation/`: test plans and release gates.
- `governance/`: documentation standards and change log.
